Protocolos

Capa 2: Enlace de Datos (Data Link Layer)

Función: Transmisión confiable de tramas en una red física, control de acceso al medio (MAC) y detección/corrección de errores.

Protocolos y estándares:

  • Ethernet (IEEE 802.3): Dominante en redes cableadas (LAN).
  • Wi-Fi (IEEE 802.11): Para redes inalámbricas (ej: 802.11ac, Wi-Fi 6).
  • PPP (Point-to-Point Protocol): Conexiones punto a punto (ej: DSL).
  • MAC (Media Access Control): Direccionamiento físico (ej: direcciones como 00:1A:2B:3C:4D:5E).
  • VLAN (IEEE 802.1Q): Segmentación de redes lógicas en switches.
  • ARP (Address Resolution Protocol): Resuelve direcciones IP a MAC.
  • HDLC (High-Level Data Link Control): Comunicación síncrona (WAN).
  • Frame Relay: Obsoleto, usado en redes de conmutación de paquetes.

Dispositivos: Switches, bridges, tarjetas de red (NIC).

Capa 3: Red (Network Layer)

Función: Enrutamiento de paquetes entre redes usando direcciones lógicas (IP).

Protocolos y estándares:

  • IP (Internet Protocol): IPv4 (ej: 192.168.1.1) e IPv6 (ej: 2001:0db8:85a3::).
  • ICMP (Internet Control Message Protocol): Diagnóstico y errores (ej: ping, traceroute).
  • IPsec (IP Security): Cifrado y autenticación de paquetes.
  • OSPF (Open Shortest Path First): Protocolo de enrutamiento dinámico.
  • BGP (Border Gateway Protocol): Enrutamiento entre sistemas autónomos (Internet).
  • RIP (Routing Information Protocol): Enrutamiento básico (obsoleto).
  • GRE (Generic Routing Encapsulation): Túneles para VPNs.

Dispositivos: Routers, firewalls de capa 3.

Capa 4: Transporte (Transport Layer)

Función: Control de flujo, segmentación y entrega confiable de datos (puertos lógicos).

Protocolos y estándares:

  • TCP (Transmission Control Protocol): Conexión confiable (ej: navegación web, correo).
  • UDP (User Datagram Protocol): Conexión no confiable (ej: streaming, VoIP).
  • SCTP (Stream Control Transmission Protocol): Para telecomunicaciones (ej: 4G/5G).
  • DCCP (Datagram Congestion Control Protocol): Mezcla de TCP/UDP (ej: streaming en tiempo real).

Ejemplos de puertos:

  • TCP 80 (HTTP), TCP 443 (HTTPS), UDP 53 (DNS), UDP 123 (NTP).

Capa 5: Sesión (Session Layer)

Función: Gestión de sesiones (inicio, mantenimiento y cierre de conexiones).

Protocolos y estándares:

  • NetBIOS: Comunicación en redes locales (legado Windows).
  • RPC (Remote Procedure Call): Llamadas a funciones remotas.
  • SIP (Session Initiation Protocol): Establece sesiones VoIP o videollamadas.
  • PPTP (Point-to-Point Tunneling Protocol): Túneles VPN (obsoleto).
  • TLS/SSL (Transport Layer Security/Secure Sockets Layer): Gestiona sesiones seguras (aunque también trabaja en capa 6).

Capa 6: Presentación (Presentation Layer)

Función: Traducción de datos (formato, cifrado, compresión).

Protocolos y estándares:

  • TLS/SSL: Cifrado de datos (ej: HTTPS).
  • JPEG/PNG/MP3: Formatos de compresión de imágenes/audio.
  • ASCII/Unicode: Codificación de texto.
  • MPEG: Codificación de video.
  • AES/DES: Algoritmos de cifrado (aplicados aquí o en capa 7).

Capa 7: Aplicación (Application Layer)

Función: Interfaz para aplicaciones de usuario y servicios de red.

Protocolos y estándares:

  • HTTP/HTTPS: Navegación web.
  • DNS (Domain Name System): Resolución de nombres a IPs.
  • SMTP/POP3/IMAP: Envío y recepción de correos electrónicos.
  • FTP/SFTP/SCP: Transferencia de archivos.
  • SNMP (Simple Network Management Protocol): Monitoreo de red.
  • SSH (Secure Shell): Acceso remoto seguro.
  • DHCP (Dynamic Host Configuration Protocol): Asignación automática de IPs.
  • Telnet: Acceso remoto sin cifrar (obsoleto).
